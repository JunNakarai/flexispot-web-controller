# Repository Guidelines

## プロジェクト構成とモジュール

アプリは `Vite + TypeScript` ベースの静的 Web サイトです。機能追加時は、UI、状態管理、シリアル通信を分離した現行構成を維持してください。
- `index.html` はアプリのエントリ。`#app` のマウントポイントとフォント読込だけを持つ前提です。
- `src/main.ts` はブートストラップ専用です。初期化処理はここに閉じ込め、画面ロジックは増やさないでください。
- `src/ui/app.ts` の `FlexiSpotApp` が UI の中核です。新しい操作や表示はまずクラスメソッドとして追加し、描画とイベント接続の責務をここに集約します。
- `src/serial/client.ts` は Web Serial API との接続管理、送受信、リピート送信を担当します。ポート制御や通信ライフサイクルの変更はここで行います。
- `src/serial/protocol.ts` はコマンドフレームと受信データのデコードを管理します。新しいコマンドやパケット形式の対応はここを更新します。
- `src/state/storage.ts` は `localStorage` 永続化を担当します。プリセットや履歴の保存仕様を変える場合はキー互換性に注意してください。
- `src/types.ts` は共通型定義です。状態追加時はここを先に更新してから各モジュールへ反映します。
- `src/styles.css` は全体テーマとレイアウトを定義します。4 スペースインデントと既存トークン設計を維持してください。
- `.github/workflows/static.yml` は GitHub Pages へのビルドと配信を担当します。Node / build 手順を変えた場合はここも追従させます。

## ビルド・テスト・開発コマンド

Web Serial API の制約上、検証は HTTPS または `localhost` で行います。
- 依存インストール: `npm install`
- 開発サーバー: `npm run dev`
- 型チェック: `npm run typecheck`
- 本番ビルド: `npm run build`
- ビルド済みプレビュー: `npm run preview`
- HTTPS スモークテストが必要な場合は、Vite の preview とは別にローカル HTTPS サーバーを用意し、手順を `README.md` に追記してください。

## コーディングスタイルと命名

TypeScript は 4 スペースインデント、セミコロン必須、メソッド名は camelCase。状態は明示的な型で管理し、文字列リテラルの追加は `src/types.ts` に寄せます。CSS も 4 スペースインデントを維持し、既存のクラス命名とデザイントークンを再利用してください。コメントは通信仕様や意図の共有に必要な箇所だけに限定し、不要なデバッグログはコミット前に削除します。

## テスト指針

現状、自動テストは未整備で、手動検証が基本です。実機または模擬デバイスで、接続、切断、上下ホールド、プリセット呼び出し、高さ表示、日次履歴、診断モニタ、エラートーストを順に確認してください。シリアルプロトコルを変更した場合は、受信バイト列と期待高さの対応を PR に明記してください。

## コミットとプルリク運用

コミットは命令形で簡潔に書き、必要なら `Fixes #15` 形式で Issue を参照します。PR には変更概要、UI 変更のスクリーンショットまたは動画、手動テスト結果、追加・変更したバイトシーケンスや保存仕様の差分を記載してください。

## セキュリティと設定上の注意

ハードウェア ID、シリアルログ、生データダンプ、証明書ファイルなどの機微情報はコミット禁止です。新規コマンドフレームや危険な挙動につながる変更は、ベンダードキュメントまたは実機検証で裏取りしてください。テスターには HTTPS 利用と、検証後にブラウザのシリアル権限を解除することを案内してください。
