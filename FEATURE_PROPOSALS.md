# 機能拡充の具体的な提案

この文書は、現状の実装を前提に「次に着手しやすい提案」を具体化したメモです。高さ表示や履歴可視化はすでに入っているため、ここでは未着手の拡張だけを扱います。

## 即座に実装可能な機能（高優先度）

### 1. 設定モーダル

**現状:**
- 設定変更用の UI がない
- テーマ、通知、診断設定、送信間隔などをユーザーが調整できない

**実装内容:**
- モーダル形式の設定画面
- テーマ選択
- 診断ログの既定動作設定
- 通知の有効 / 無効
- 高度な設定として送信間隔や自動切断時間を保持

**実装ファイル:**
- `src/ui/app.ts`
- `src/state/storage.ts`
- `src/types.ts`
- `src/styles.css`

**メリット:**
- 設定の散在を防げる
- 以後の機能追加で UI の置き場に困らない
- PWA 化や通知機能の土台になる

---

### 2. プリセット管理の拡張

**現状:**
- 表示名の変更はできる
- 保存高さや任意プリセットの追加はできない

**実装内容:**
- 現在高さをプリセット候補として保存
- プリセットの編集・削除
- 並び順変更
- 将来の複数デスク対応を見据えた保存形式への拡張

**実装ファイル:**
- `src/ui/app.ts`
- `src/state/storage.ts`
- `src/types.ts`

**メリット:**
- いまの UI を壊さず体感価値を増やしやすい
- 保存データ構造を先に整えることで後続機能が楽になる

---

### 3. アクセシビリティ改善

**現状:**
- キーボードショートカットはある
- ただしフォーカス管理、ARIA、チャートやトーストの補助説明は不足している

**実装内容:**
- 主要ボタンへの `aria-label` と状態説明
- トーストと診断領域の読み上げ改善
- モーダル導入時のフォーカストラップ
- 色だけに依存しない状態表現

**実装ファイル:**
- `src/ui/app.ts`
- `src/styles.css`

**メリット:**
- ハードウェア操作 UI として安全性が上がる
- 将来のレビューや公開時に説明しやすい

---

## 中期的に実装すべき機能

### 4. テスト自動化

**実装内容:**
- `src/serial/protocol.ts` のデコードテスト
- `src/state/storage.ts` の保存・トリムロジックのテスト
- 主要 UI のスモークテスト

**候補技術:**
- Vitest
- Playwright

**メリット:**
- 実機でしか気づけない破損を減らせる
- プロトコル変更に踏み込みやすくなる

---

### 5. PWA 対応

**実装内容:**
- Web App Manifest
- Service Worker による静的アセットキャッシュ
- インストール導線

**実装ファイル候補:**
- `index.html`
- `src/` 配下のエントリ周辺
- `public/` ではなく Vite の静的アセット配置に合わせたファイル追加

**メリット:**
- デスクトップアプリに近い導線を作れる
- GitHub Pages 配信との相性が良い

---

### 6. 履歴データの強化

**実装内容:**
- 週次 / 月次の可視化
- CSV / JSON エクスポート
- `IndexedDB` への移行

**メリット:**
- 今の 14 日 / 1 日 1440 件制限を超えられる
- 健康管理用途に広げやすい

---
